### 失敗と再発防止
| 失敗 | 根本原因 | 再発防止策 |
|------|----------|------------|
| Worktree merge conflicts (Wave 3) | Worktree agents started from pre-Wave-2 base, diverged on shared files | Use `git show` + manual file extraction instead of `git merge` for worktree changes |
| Worktrees added to git as submodules | `git add -A` includes .claude/worktrees/ | Clean up worktrees BEFORE committing, or add `.claude/worktrees/` to .gitignore |
| uv.sources missing for workspace dep | Plan didn't include `[tool.uv.sources]` section | Always include `tool.uv.sources` when adding workspace deps |
| mypy can't find workspace deps | mypy doesn't resolve uv workspace deps automatically | Add `mypy_path` to root pyproject.toml for all src dirs |
| click.DateTime shows format not metavar | click.DateTime uses format string as help text, not "BIRTHDATE" | Test for `%Y-%m-%d` in help output instead of argument name |
| taiun gender type annotation wrong | Plan used `click.Parameter` instead of `Gender` | Use actual domain type (Gender) for click custom type return values |

### 学び
- For monorepo mypy: `mypy_path` in root config must include all package src dirs
- uv workspace requires explicit `[tool.uv.sources]` in consuming packages
- Worktree parallel agents: prefer `git show` extraction over `git merge` for small, targeted changes
- click.DateTime metavar shows format string, not argument name — adjust tests accordingly
- Wave 4 (sequential tasks modifying shared files) was more efficient than worktree parallelization would have been
